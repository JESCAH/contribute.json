What is this?

It's an project to find a common standard to describe all active Mozilla websites and projects.

Two primary use cases:

  • Ability for a third-party service to pull in all of these and make a listing or dashboard.
  • Ability for an individual to find out more about a website or project by simply looking for the /contribute.json URL.

What are the use cases?

In no particular order...

  • You have a skill, say AngularJS or C++, but don't know which projects use this. If you could collect all the keywords from all projects using contribute.json you could easily find out which projects to approach with your friendly skill.
  • You're browsing a site and discover that something is broken. You could just close the tab in anger or you can help out in some ways. Perhaps a good place is to file a bug report. Or find out where the code is and go there directly and make a patch. Or just find out which IRC channel, wiki page or mailing list people on the project hang out on.
  • You're a developer asked to help on a project and you set to work. Instead of asking all the usual questions like "Where is the code?", "Where is the staging server?", "Do we have a mailing list?" you can simply look all this up in the /contribute.json file.
